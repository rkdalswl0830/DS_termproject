# EDA 해석

데이터: OSMI Mental Health in Tech Survey 2014, 1,259명 × 27개 변수
타겟: `treatment` (정신건강 치료 경험 여부)

---

## 2. 기본 검토

**목적**: 변수 구성과 데이터 타입을 파악해 전처리 방향 결정.

**패턴**
- 수치형 1개(`Age`), 범주형 26개
- `Age` 통계가 비정상 (min -1726, max 1e11) → 입력 오류 존재
- `Gender` unique 49개, `Country` 미국 60% 편향

**해석**: Age 극단치 제거, Gender 표준화, Country 그룹화가 전처리 핵심.

---

## 3. 결측치 분석

**목적**: 변수별 결측 비율을 확인해 삭제/대체/유지 결정.

**패턴**
- `comments` 87%, `state` 41%, `work_interfere` 21%, `self_employed` 1.4%

**해석**
- `comments`(자유서술), `state`(미국 한정) → 삭제
- `work_interfere` 결측은 "정신건강 이슈 없음"과 연관된 비무작위 결측 → `"None"` 별도 범주로 처리
- `self_employed` → 최빈값 대체

---

## 4. 타겟 변수 (`treatment`)

**목적**: 클래스 불균형 여부 확인.

**패턴**: Yes 50.6% / No 49.4%

**해석**: 거의 완벽한 균형 → 재샘플링 불필요, 이진 분류에 이상적.

---

## 5. `Age` 변수

**목적**: 유일한 수치형 변수의 분포와 극단치 확인.

**패턴**
- 원본에 -1726, 99999999999 등 8개 비현실적 값
- 18-75세 필터링 후 평균 32세, 중앙값 31세, 약한 우편향

**해석**: 응답자가 IT업계 일반 분포(20-30대)와 일치. 극단치 8건 제거 후 사용.

## 6-1. `Gender` (가장 정제 필요)

**목적**: 표기 변형 확인 후 표준화 규칙 결정.

**변수 선택 이유**: 성별은 치료율 차이가 크게 알려진 변수이므로 정제 필수.

**패턴**: unique 49개. `Male`/`male`/`M`/`m`/`Make`/`Mal` 등 동의어 다수, 트랜스/논바이너리 응답, `Nah`/`p` 같은 무의미 응답 존재.

**해석**: `Male` / `Female` / `Other` 3개로 표준화 권장.

---

## 6-2. `Country`

**목적**: 국가 분포 확인, 카디널리티 처리 방향 결정.

**변수 선택 이유**: 국가별 의료 접근성·문화가 다르므로 맥락 변수로 의미 있음.

**패턴**: 총 48개국, 미국 60% 압도, 응답 ≤3명 국가가 28개.

**해석**: 상위 5-6개국만 유지, 나머지 `Other`로 통합. 분석 결과는 사실상 "미국 IT 종사자" 경향임을 보고서에 명시 필요.

## 6-3. 범주형 변수 의미 그룹 분류

**목적**: 27개 변수를 의미 단위로 묶어 가설 수립.

**4개 그룹**: 회사 정보 / 정신건강 제도 / 직장 분위기 / 개인 배경

**주요 패턴**
- `wellness_program` No 67% — 대부분 회사가 미운영
- `anonymity` Don't know 65% — 익명성 보장 여부조차 모름
- `family_history` Yes 39%

**해석**: 제도는 있어도 직원이 인지·신뢰하지 못함. 사회적 낙인 여전. family_history가 강한 예측 변수일 가능성 시사.

## 7-1. Mental Health Policy vs Treatment

**목적**: 회사 **제도**가 직원 치료 행동과 어떻게 연관되는지 정량화.

**패턴 (treatment=Yes 비율)**
| 변수 | 가장 낮음 → 가장 높음 | 차이 |
|---|---|---|
| **care_options** | No 41% / Not sure 39% / Yes **69%** | +28%p |
| **benefits** | Don't know 37% / No 48% / Yes **64%** | +27%p |
| leave (어려움) | 휴가 쉬움 50% / 매우 어려움 **68%** | +18%p |
| wellness_program | 43% / 50% / 60% | +17%p |
| anonymity | 45% / 58% / 62% | +17%p |
| seek_help | 45% / 50% / 60% | +15%p |

**해석**
- **`care_options=Yes`인 직원의 69%가 치료 경험** → 진료 옵션 제공이 가장 강한 정책 효과
- **`benefits=Yes`인 직원의 64%가 치료 경험** → 보험 혜택이 실제 치료로 이어짐
- `leave=Very difficult` 그룹의 치료율이 높은 건 "심각한 직원이 휴가 어려운 환경에 있다"는 역인과 해석이 타당
- `Don't know` 응답군이 일관되게 가장 낮음 → **제도 인지 부족이 곧 치료 장벽**

---

## 7-2. Workplace Atmosphere vs Treatment

**목적**: **분위기/문화**(제도 아닌 인적 요인)의 영향 측정.

**패턴 (treatment=Yes 비율)**
| 변수 | 패턴 | 차이 |
|---|---|---|
| **obs_consequence** | No 48% / Yes **70%** | +22%p |
| mental_health_interview | Maybe 40% / Yes 60% | +20%p |
| mental_health_consequence | No 43% / Yes 60% | +17%p |
| coworkers | 46% / 50% / 57% | +11%p |
| supervisor | 47% / 51% / 49% | ≈ 0%p |

**해석**
- `obs_consequence=Yes`(동료 불이익 목격) 70% 치료율 → **이미 치료받은 사람일수록 부정적 사례에 민감** (역인과 해석)
- `supervisor`는 거의 효과 없음 → 상사 대화 가능 여부가 실제 행동과 무관
- `coworkers`는 약한 양의 상관 → 동료 지지는 도움 되지만 결정적이지 않음

---

## 7-3. Personal Factors vs Treatment ☆ 핵심

**목적**: 회사가 통제 불가능한 **개인 요인**의 영향 측정.

**패턴 (treatment=Yes 비율)**
| 변수 | 패턴 | 차이 |
|---|---|---|
| **family_history** | No **35%** / Yes **75%** | **+40%p ☆☆** |
| **work_interfere** | Never 15% / Rarely 70% / Sometimes 77% / Often **85%** | +70%p |
| self_employed | 50% / 53% | ≈ 0%p |

**해석**
- **`family_history=Yes` 응답자의 75%가 치료 경험. No인 사람은 35% → +40%p 격차로 가장 강력한 단일 예측 변수**
- `work_interfere`는 극명한 단조 증가지만 증상 심각도 자체와 거의 동치 → **데이터 누출(leakage) 위험**, 모델 사용 시 신중
- `self_employed`는 거의 무관, 제거 고려 가능

---

## 7-4. Age vs Treatment

**목적**: 연령이 치료 결정에 영향을 주는지 확인.

**패턴**
- No 그룹 중앙값 30세, Yes 그룹 32세
- 두 분포가 거의 겹침

**해석**: **연령은 거의 영향 없음**. 통념과 달리 IT 업계 내 연령차는 미미. 응답자가 20-30대 집중인 점도 영향. 단독으로는 약하나 다른 변수와의 교호작용에서 의미 가질 수 있음.

---

## 7-5. Gender vs Treatment

**목적**: 성별별 치료율 차이 확인 (여성이 남성보다 적극적으로 치료받는 경향이 일반적).

**예상 해석**: 여성 치료율이 높게 나온다면 **남성의 도움 요청 장벽이 크다**는 기존 연구와 일치.
